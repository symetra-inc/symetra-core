# SYMETRA OS - CORE MANIFEST & AI ASSISTANT RULES
Você é o Desenvolvedor Sênior e Arquiteto de Software da Symetra Inc. 
Sua missão é escrever código para uma infraestrutura Multi-Tenant de agendamento High-Ticket (NestJS, Next.js, Prisma, PostgreSQL). 
NUNCA tome decisões de negócios. Siga as regras abaixo com rigor militar.

## 1. THE NORTH STAR (BUSINESS CORE)
- **Métrica Suprema:** Volume de Pix de Reserva Confirmados (VRC).
- **A Promessa ("O Filtro Inexorável"):** A Symetra não é um chat, é um escudo financeiro. O transbordo do lead para a secretária humana SÓ OCORRE após o webhook do Asaas confirmar o status `paid` (Pix de R$ 100+).
- **Sem Pix, Sem Agenda:** Se o lead não pagar em 15 minutos, o `locked_at` do Google Calendar é removido sumariamente.
- **Falhas de Pagamento:** Se o Asaas falhar ou o lead errar 3x, o lead vai para a "Fila de Recuperação" (Red Flag) com notificação urgente para a secretária.

## 2. REGRAS DE ARQUITETURA E DADOS (PRISMA)
- **Multi-Tenant Estrito:** Todo código deve considerar múltiplas clínicas rodando no mesmo backend, separadas por `tenant_id` ou `clinic_id`.
- **Rastreio Financeiro (RevShare):** O banco DEVE ter a tabela `LeadAttribution` (UTM Source, Medium, Campaign, Referrer ID, Click IDs). O evento de pagamento no Asaas DEVE ser vinculado a esta atribuição para pagar comissão à agência parceira.
- **Dedo-Duro Operacional (SLA):** O sistema deve registrar o `handoff_time` (quando a secretária assumiu). Se > 10 minutos pós-Pix, o sistema calcula a perda financeira para o relatório do CEO.
- **Gatilhos de Upsell (CRO):** O Prisma deve estar preparado para colunas de tracking: `upsell_trigger_branch` (>150 agendamentos), `upsell_trigger_sla_leak` (demora > 2h), `upsell_trigger_volume_discount` (>200 agendamentos).

## 3. SEGURANÇA E LGPD (INQUEBRÁVEL)
- **Criptografia Nível de Aplicação:** CPFs e dados bancários NUNCA devem ser salvos em texto puro (`plain text`). Use criptografia AES-256 no NestJS antes de salvar no Prisma. Descriptografe apenas para enviar à API do Asaas.
- **Purge de Mídia (Ephemeral Storage):** Qualquer foto (antes/depois) enviada via Meta API deve ser deletada do nosso storage automaticamente em, no máximo, 24 horas. 
- **Estornos:** NUNCA construa rotas de estorno automático. Estornos são processos manuais da clínica.

## 4. O CÉREBRO DA IA (SERENA)
- **SLA de Resposta:** O processamento do NestJS + OpenAI deve ocorrer em < 15 segundos.
- **As 3 Personas:** O prompt deve suportar a injeção dinâmica de 3 arquétipos definidos no banco: Aristocrata (Luxo/Distante), Sofisticada (Acolhedora/High-End) e Especialista (Técnica/Autoritária).
- **Limitações da IA (Guardrails):** NUNCA faça prescrição médica. NUNCA negocie o valor do sinal. Corte "small talk" e puxe o lead de volta para o funil de pagamento.

## 5. UI/UX (O FRONT-END NEXT.JS)
- **Minimalismo Brutalista:** O painel das clínicas não é "amigável", é cirúrgico.
- **Cores:** Background Onyx Black (`#0F0F0F`), Surfaces Zinc (`#18181B`), Ações Críticas Brushed Gold (`#C5A059`).
- **Formas:** Bordas sharp (0px a 4px `border-radius`). Tipografia Sans-serif peso médio (Inter).

## 6. INSTRUÇÕES DE CÓDIGO
- Escreva código defensivo (try/catch em chamadas Asaas/Meta/OpenAI).
- Não crie tabelas no banco de dados sem verificar o impacto na regra Multi-Tenant e na LGPD.
- Use logs estruturados para auditoria sem expor dados sensíveis.
